8    * In order to import key-value pairs to TiKV, the user should:
9    * 1. Open an engine identified by an UUID.
10   * 2. Open write streams to write key-value batches to the opened engine.
11   *    Different streams/clients can write to the same engine concurrently.
12   * 3. Close the engine after all write batches have been finished. An
13   *    engine can only be closed when all write streams are closed. An
14   *    engine can only be closed once, and it can not be opened again
15   *    once it is closed.
16   * 4. Import the data in the engine to the target cluster. Note that
17   *    the import process is not atomic, it requires the data to be
18   *    idempotent on retry. An engine can only be imported after it is
19   *    closed. An engine can be imported multiple times, but can not be
20   *    imported concurrently.
21   * 5. Clean up the engine after it has been imported. Delete all data
